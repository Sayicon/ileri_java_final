## Proje Özeti

Kocaeli Üniversitesi BSM TBL324 İleri Java Uygulamaları dersi kapsamında geliştirilen bu proje, dersin **dağıtık sistem · OOP · veri katmanı · test** kazanımlarını tek bir bütünleşik Java uygulamasında ölçer.

**Hedef:** 65 puan zorunlu iskelet + 35 puan tüm ek özellikler = **100 puan**. Proje **mikroservis** olarak inşa edilir (4 servis: Auth · Event · Ticket · Notification).

**Tema:** Etkinlik Bileti & Salon Yönetim Sistemi (Event Ticketing).

**Ekip:** Kerem + Efe. Commit dağılımı ~50/50 hedef (NOT-2: orantısız commit → 0). Sunumda her modül her ikisine sorulabilir — pair-walkthrough her faz sonunda yapılmalı.

**Kısa teknoloji özeti:** Java 21 · Spring Boot 3.3 · Saf JDBC + PostgreSQL · Redis (Jedis) · JavaFX 21 · Android native Java · Spring Cloud Gateway · Docker Compose · JUnit 5 + Testcontainers · k6+JMeter. Detay → [`docs/plan/tech-stack-and-rubric.md`](docs/plan/tech-stack-and-rubric.md)

## Mimari İlkeler

### OOP & SOLID Yansıması

Tüm SOLID prensipleri uygulanır. **Design Patterns:** Strategy (PaymentStrategy, Notifier), Factory (NotificationFactory), Repository (Generic JDBC), DTO↔Entity Mapper, Template Method (BaseJdbcRepository), Builder (Lombok), Singleton (Spring bean).

### Generic Yapılar (10 puan)

`ApiResponse<T>`, `PagedResult<T>`, `Repository<T, ID>`, `Validator<T>` — hepsi işlevsel, dekoratif değil. Wildcards (`? extends`, `? super`) en az 2 yerde (`CollectionOps.java`).

### Veri Katmanı

Saf JDBC + manuel `ResultSet → Entity` mapping. HikariCP pool. `@Transactional` Spring AOP. Flyway migration. Redis: session/lock/rate-limit için **birincil** depo (cache değil). Key pattern: `session:{userId}`, `lock:seat:{eventId}:{seatId}`, `ratelimit:{ip}:{endpoint}`.

### Hata Yönetimi (5 puan)

`@ControllerAdvice` + RFC 7807 ProblemDetail. 400/401/403/404/409/429/500/503 standart haritası. Her servis aynı kuralları uygular.

### TDD (+10 puan)

Her fazda: **A commit** (testler kırmızı) → `test-logs/faz-N-red.txt` → **B commit** (uygulama, testler yeşil) → `test-logs/faz-N-green.txt`. Commit timestamp kanıt. Mesaj formatı: `test(fazN): ...` / `feat(fazN): ...`.
